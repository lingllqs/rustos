## 实模式下内存布局

| 起始地址 | 结束地址 | 大小       | 用途               |
|----------|----------|------------|--------------------|
| 0x000    | 0x3ff    | 1KB        | 中断向量表         |
| 0x400    | 0x4ff    | 256B       | BIOS 数据区        |
| 0x500    | 0x7bff   | 29.75KB    | 可用区域           |
| 0x7c00   | 0x7dff   | 512B       | MBR 加载区域       |
| 0x7e00   | 0x9fbff  | 607.6KB    | 可用区域           |
| 0x9fc00  | 0x9ffff  | 1KB        | 扩展 BIOS 数据区   |
| 0xa0000  | 0xaffff  | 64KB       | 用于彩色显示适配器 |
| 0xb0000  | 0xb7fff  | 32KB       | 用于黑白显示适配器 |
| 0xb8000  | 0xbffff  | 32KB       | 用于文本现实适配器 |
| 0xc0000  | 0xc7fff  | 32KB       | 用于适配器 BIOS    |
| 0xc8000  | 0xeffff  | 160KB      | 映射内存           |
| 0xf0000  | 0xfffef  | 64KB - 16B | 系统 BIOS          |
| 0xffff0  | 0xfffff  | 16B        | 系统 BIOS 入口地址 |

## BIOS 内存检测

**使用 0x15 BIOS 中断, 功能号为 0xe820**

- 存放内存表位置为 es:di
- ebx 必须置为0, 检测过程改变
- eax 存放功能号 0xe820
- ecx 存放结构体大小

**0x15 中断, 功能号 0xb820, 返回结果**
- eax 内容为 0x534d4150 - SMAP 的 ascii 码
- ebx 内容为内存数量, 为0表示结束

**BIOS 返回的内存信息数据结构(20字节)**

```c
struct {
    uint64_t base;   // 起始地址
    uint64_t length; // 长度
    uint32_t type;   // 类型
    uint32_5 attr;   // 扩展(新)
};
```

| type | 含义     |
|------|----------|
| 1    | 可用内存 |
| 2    | 保留     |
| 3    | ACPI     |
| 4    | 保留     |
| 5    | 坏内存   |
